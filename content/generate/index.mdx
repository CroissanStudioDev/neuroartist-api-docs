# Каталог моделей

Все доступные модели — image, video, audio — лежат в одном каталоге. Чтобы вызвать модель, вам достаточно знать её `modelId`. Никаких отдельных подключений и SDK на каждое семейство.

## Что такое модель в каталоге

Карточка модели — это объект с фиксированным набором полей:

- **`modelId`** — публичный идентификатор без префиксов. Используется в `POST /run/{modelId}` и `POST /queue/{modelId}`. Не меняется при обновлениях модели.
- **`category`** — тип задачи: `text-to-image`, `image-to-image`, `text-to-video`, `image-to-video`, `text-to-speech`, `speech-to-text`, …
- **`schema`** — JSON Schema того, что модель принимает на вход. Точные поля, типы, enum-ы, default-ы.
- **`priceRub`** — цена за единицу (image, second, megapixel, …). Decimal, может быть меньше 1 ₽.
- **`creditCost`** — сколько спишется при вызове. Integer, минимум 1 ₽.
- **`tags`** — для поиска: `realism`, `lora`, `multilingual`, `nsfw-safe`, …

Полное описание полей — [схема модели](/generate/schema).

## Как ведёт себя каталог

- **Стабильный `modelId`.** Идентификатор не меняется. Если модель переименовывается — добавляется новый `modelId`, старый какое-то время остаётся работающим.
- **Только модели с ценой.** В публичном каталоге видны только модели с синхронизированной ценой и `visible=true`. Модель появляется в `GET /models` сразу после публикации цены.
- **Цены динамические.** `priceRub` — текущая цена за единицу с учётом скидок. `creditCost` — это интеджерная стоимость списания (минимум 1 ₽). Подробнее — [схема модели](/generate/schema#цена).
- **modelId со слешами.** Допустимо: `flux/dev`, `kling-video/v2/standard/text-to-video`. Передавайте как есть, не URL-кодируйте.
