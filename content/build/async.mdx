# Асинхронная очередь и SSE

Submit возвращает `requestId` сразу. Прогресс — через SSE.

## Полный код

```js
import { fetchEventSource } from '@microsoft/fetch-event-source'

const NA_API_KEY = process.env.NA_API_KEY
const BASE = 'https://api.neuroartist.ru'

async function runAsync(modelId, input) {
  // 1. Submit
  const submit = await fetch(`${BASE}/queue/${modelId}`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${NA_API_KEY}`, 'Content-Type': 'application/json' },
    body: JSON.stringify(input)
  }).then(r => r.json())

  if (!submit.requestId) throw new Error(`Submit failed: ${submit.message ?? submit.code}`)

  // 2. Wait via SSE
  return new Promise((resolve, reject) => {
    fetchEventSource(
      `${BASE}/queue/${modelId}/requests/${submit.requestId}/progress/stream`,
      {
        headers: { Authorization: `Bearer ${NA_API_KEY}` },
        onmessage(msg) {
          const e = JSON.parse(msg.data)
          if (e.stage === 'completed') resolve(e.result)
          if (e.stage === 'failed') reject(Object.assign(new Error(e.error?.message), e.error))
        },
        onerror(err) { reject(err) }
      }
    )
  })
}

const result = await runAsync('flux/dev', {
  prompt: 'editorial portrait',
  image_size: 'portrait_4_3'
})
console.log(result.images[0].url)
```

## Пайплайн

```
POST /queue/{modelId}                          → { requestId }
GET  /queue/{modelId}/requests/{id}/progress/stream  ← SSE: queued → in_progress → completed
GET  /queue/{modelId}/requests/{id}            → результат (можно перечитать)
PUT  /queue/{modelId}/requests/{id}/cancel     → отмена с refund
```

## Polling вместо SSE

Если SSE недоступен (serverless с лимитом 30 сек):

```bash
while true; do
  STATUS=$(curl -s "https://api.neuroartist.ru/queue/flux/dev/requests/req_abc/status" \
    -H "Authorization: Bearer na_live_xxxx" | jq -r .status)
  [[ "$STATUS" == "COMPLETED" || "$STATUS" == "FAILED" ]] && break
  sleep 2
done

curl "https://api.neuroartist.ru/queue/flux/dev/requests/req_abc" \
  -H "Authorization: Bearer na_live_xxxx"
```

Расход — 1 квота API-ключа на каждый poll.

## Webhook вместо SSE

Самый устойчивый путь для серверных pipeline-ов:

```bash
curl -X POST 'https://api.neuroartist.ru/queue/flux/dev?webhook=https://your.app/cb' \
  -H "Authorization: Bearer na_live_xxxx" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"…"}'
```

После terminal-стадии сервис POST-ит подписанный JSON на ваш URL. → [готовый приёмник webhook](/build/webhooks).

## Параллельные запросы

```js
const ids = await Promise.all([
  submit({ prompt: 'red flower' }),
  submit({ prompt: 'blue flower' }),
  submit({ prompt: 'yellow flower' })
])
const results = await Promise.all(ids.map(waitForCompletion))
```

Лимит SSE-стримов — `SSE_MAX_CONNECTIONS_PER_USER` (default 5).

## Подключение после завершения

Можно подключиться к `progress/stream` **после** terminal-стадии — сервис отдаст последнее событие и закроет стрим. Удобно для UI после refresh с сохранённым `requestId`.

## Гибридный паттерн для production

- **Submit + webhook** — серверный path (idempotent, переживает рестарты).
- **SSE** — UX в браузере, опционально, поверх того же `requestId`.

## Отмена

```bash
curl -X PUT https://api.neuroartist.ru/queue/flux/dev/requests/req_abc/cancel \
  -H "Authorization: Bearer na_live_xxxx"
```

Деньги возвращаются (refund-транзакция).
