# Хранение результатов

После завершения генерации gateway автоматически копирует артефакты из CDN провайдера в наш S3 (Cloud.ru) и переписывает URL-ы в payload. Клиенту менять ничего не нужно.

## Зачем

CDN провайдеров — temporary storage (~24 часа, иногда меньше). У вас в БД будут битые ссылки. Наш S3:

- Долговременные ссылки.
- GDPR-friendly — можно удалить всё для конкретного юзера.
- Стабильный CDN с одинаковыми latency.

## Что переписывается

- `images[].url`, `image.url`
- `video.url`
- `audio.url`
- Всё, что определено в provider-specific URL-extractor

`seed`, `timings`, `metadata` — оставляем как есть.

## Где работает

| Ручка | Когда переписывает |
| --- | --- |
| `POST /run/{modelId}` | Перед ответом |
| `GET /queue/.../requests/{id}` | Перед выдачей результата |
| Webhook → ваш URL | Перед forwarding |

## Bypass — passthrough

Получить **исходные** URL провайдера:

```bash
curl 'https://api.neuroartist.ru/run/fal-ai/flux/dev?passthrough=1' …
curl 'https://api.neuroartist.ru/queue/fal-ai/flux/dev/requests/req_abc?passthrough=1' …
```

Используйте только для отладки — ссылки умрут через несколько часов.

## Если upload упал

Любая ошибка S3 (5xx, таймаут, > `ASSET_MAX_BYTES` = 500 MB) → graceful fallback: оставляем CDN-URL провайдера. Запрос **не падает**, генерация считается успешной.

В `/me/activity/{requestId}` запись со статусом `failed` для конкретного asset-а. Скачайте URL к себе пораньше.

## Лимиты

| Параметр | Значение |
| --- | --- |
| `ASSET_MAX_BYTES` | 500 MB / артефакт |
| Allowed MIME | `image/*`, `video/mp4|quicktime|webm`, `audio/*` |

## Удаление

Удалить весь реквест и его assets:

```bash
curl -X DELETE https://api.neuroartist.ru/me/activity/{requestId}/assets \
  -H "Authorization: Bearer na_live_xxxx"
```

Удаление необратимо — скачайте, если нужны архивы.
