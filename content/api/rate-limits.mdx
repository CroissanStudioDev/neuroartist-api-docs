# Лимиты запросов

Все лимиты sliding-window поверх Redis. Превышение → `429 Too Many Requests`:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1714050300
Retry-After: 47
```

## Все лимиты

| Область | Лимит | На что |
| --- | --- | --- |
| IP | 10 / 5 мин | `POST /api/auth/sign-in/email` |
| IP | 5 / 5 мин | `POST /api/auth/sign-up/email` |
| IP | 3 / 10 мин | `POST /api/auth/forgot-password/email` |
| IP | 100 / 1 мин | `POST /webhooks/*` |
| User | `USER_GENERATION_LIMIT_PER_HOUR` (default 100) / час | `POST /run/*`, `POST /queue/*`, `POST /integrations/*` |
| API-key | 1000 / час | Любая ручка через ключ |
| API-key | `metadata.monthlyLimitRub` (опц.) | Расход в ₽/месяц на ключ → `429 key_limit_exceeded` |
| User | `SSE_MAX_CONNECTIONS_PER_USER` (default 5) | Одновременные SSE-стримы |

## Retry-стратегия

```js
async function callWithRetry(url, init, maxRetries = 3) {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    const resp = await fetch(url, init)
    if (resp.status !== 429 && resp.status !== 503) return resp
    if (attempt === maxRetries) return resp
    const retryAfter = Number(resp.headers.get('Retry-After')) || 1
    await new Promise(r => setTimeout(r, retryAfter * 1000))
  }
}
```

`503 provider_error` тоже отдаёт `Retry-After` — это circuit breaker, провайдер флапает.

## Месячный лимит на ключ

```bash
curl -X PATCH https://api.neuroartist.ru/me/keys/key_abc \
  -H "Content-Type: application/json" -b cookies.txt \
  -d '{"metadata":{"monthlyLimitRub":5000}}'
```

Используйте, чтобы:
- Ограничить ключ для интеграции (не больше N ₽/мес).
- Дать sandbox-ключ с маленьким cap для тестов.
- Защититься от непредвиденного расхода в production.

Сбрасывается 1 числа каждого месяца.

## Что НЕ ограничено

- `GET` ручки (`/models`, `/me/balance`, `/me/transactions`) — только глобальный 1000/час.
- `/health`, `/openapi.json`, `/docs` — публичные.

## Заголовки в CORS

`X-RateLimit-*` и `Retry-After` — в expose-list, читаемы из браузера.
