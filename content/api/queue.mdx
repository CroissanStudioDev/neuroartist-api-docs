# `/queue`

Async-режим: submit возвращает `requestId` сразу, terminal-состояние получаете через polling или webhook.

Концептуальный гайд — [Async + polling](/build/async).

## `POST /queue/{modelId}`

Отправить задачу.

**Аутентификация:** API-key или session.  
**Параметр path:** `modelId` (со слешами, без URL-кодирования).  
**Параметры query:**

| Параметр | Описание |
| --- | --- |
| `webhook` | URL для callback после завершения |

**Тело запроса:** input модели.

```bash
curl -X POST 'https://api.neuroartist.ru/queue/flux/dev?webhook=https://your.app/api/cb' \
  -H "Authorization: Bearer na_live_xxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"editorial portrait"}'
```

**Ответ 200:**

```json
{
  "requestId": "req_abc",
  "status": "IN_QUEUE",
  "queuePosition": 1
}
```

`status` ∈ {`IN_QUEUE`, `IN_PROGRESS`} — задача в работе или ещё ждёт слот.

**Ошибки:** см. `/run` — те же.

## `GET /queue/{modelId}/requests/{requestId}/status`

Опрос статуса (без полного payload).

**Параметры query:** `logs=1` (опционально) — добавить логи генерации.

**Ответ 200:**

```json
{
  "state": "in_progress",
  "queue_position": 0,
  "logs": [ { "level": "info", "message": "Loading model", "timestamp": "…" } ]
}
```

`state` ∈ {`in_queue`, `in_progress`, `completed`, `failed`}.

**Ошибки:** `404 not_found`.

## `GET /queue/{modelId}/requests/{requestId}`

Забрать результат (после `completed`).

**Параметры query:** `passthrough=1` — оригинальные внутренние URL без re-host в наш S3.

**Ответ 200:**

```json
{
  "request_id": "req_abc",
  "images": [
    { "url": "https://s3.cloud.ru/…", "width": 768, "height": 1024, "content_type": "image/webp" }
  ],
  "seed": 12345,
  "timings": { "inference": 4.21 }
}
```

Можно вызывать **многократно** — payload сохраняется в БД.

**Ошибки:** `404 not_found` (если не `completed` ещё или request принадлежит другому пользователю).

## `PUT /queue/{modelId}/requests/{requestId}/cancel`

Отменить незавершённую генерацию.

**Ответ 200:**

```json
{ "ok": true }
```

Деньги возвращаются (refund-транзакция).

**Ошибки:** `404 not_found` (если уже завершилась — отмена бессмысленна).

## Поведение IDOR

Все ручки `/{requestId}/*` проверяют, что реквест принадлежит вам. Чужие реквесты вернут `404 not_found` — мы намеренно не различаем «нет такого» и «есть, но не ваш», чтобы не дать перебирать ID.

## Serverless / edge рантаймы

Если ваш runtime не может держать долгое polling-соединение — используйте **webhook**:

```bash
curl -X POST 'https://api.neuroartist.ru/queue/flux/dev?webhook=https://your.app/cb' \
  ...
```

После terminal-состояния сервис POST-ит JSON на ваш URL. Подробнее — [формат webhooks](/build/webhooks).
