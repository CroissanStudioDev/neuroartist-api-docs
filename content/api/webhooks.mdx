# `/webhooks`

<Callout type="info">
  Это **входящие** webhooks — внутренние server-to-server. Клиентский код к ним обращаться не должен. Если вы интегрируете API в свой продукт, вам нужны **исходящие** webhooks (от нас → ваш приёмник) — см. [приём webhook](/build/webhooks) и [формат webhooks](/build/webhooks).
</Callout>

## `POST /webhooks/yookassa/notify`

Webhook от YooKassa о состоянии платежа.

**Защита:** IP-whitelist (CIDR YooKassa) + HMAC.

**Тело запроса** (формат YooKassa):

```json
{
  "event": "payment.succeeded",
  "object": {
    "id": "2c…",
    "status": "succeeded",
    "amount": { "value": "1000.00", "currency": "RUB" },
    "metadata": { "userId": "u_…", "creditsToGrant": "1000" },
    "payment_method": { "id": "pm_abc", "type": "bank_card", "card": { "last4": "4242" } }
  }
}
```

**Ответ 200:** `{ "ok": true, "processed": true }` — или `{ "processed": false, "reason": "duplicate" }` при дедупе.

**Идемпотентность:** Redis SETNX по `webhook_id` с TTL 1 час.

**Что делает:**

- На `payment.succeeded`: зачисляет средства на баланс, сохраняет карту для повторных платежей (если ещё нет), отправляет email-нотификацию.
- На `payment.canceled`: помечает платёж `canceled`, деньги не зачисляются.

**Ошибки:** `400 invalid_json` / `invalid_body`. Подпись/IP — игнорируются (200 OK с `processed: false, reason: …`), чтобы YooKassa не ретраила бесконечно.
