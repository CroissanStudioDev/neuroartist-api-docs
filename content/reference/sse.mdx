# SSE-прогресс

Async-режим (`POST /queue`) возвращает `requestId` сразу. Чтобы получить terminal-результат push-ом, а не опросом — подписывайтесь на SSE.

## Эндпоинт

```
GET /queue/{modelId}/requests/{requestId}/progress/stream
```

Аутентификация: API-key или session (через `credentials: 'include'` в браузере).

## Формат фрейма

```
data: {"stage":"in_progress","requestId":"req_abc","timestamp":"…","payload":{…}}

```

`stage` ∈:

| Стадия | Когда |
| --- | --- |
| `queued` | Задача принята, ждём слот у провайдера |
| `in_progress` | Провайдер начал |
| `uploading_assets` | Артефакты выгружаются в S3 |
| `reconciling` | Списание корректируется по факту |
| `completed` | Готово, payload в `event.result` |
| `failed` | Ошибка, причина в `event.error` |

При `completed`:

```json
{
  "stage": "completed",
  "requestId": "req_abc",
  "result": {
    "images": [{"url":"https://s3.cloud.ru/…","width":768,"height":1024}],
    "seed": 12345
  }
}
```

При `failed`:

```json
{ "stage": "failed", "requestId": "req_abc", "error": { "code": "provider_error", "message": "…" } }
```

## Подключение

```js
import { fetchEventSource } from '@microsoft/fetch-event-source'

await fetchEventSource(
  `https://api.neuroartist.ru/queue/${encodeURIComponent(modelId)}/requests/${requestId}/progress/stream`,
  {
    headers: { Authorization: `Bearer ${process.env.NA_API_KEY}` },
    onmessage(msg) {
      const e = JSON.parse(msg.data)
      if (e.stage === 'completed') resolve(e.result)
      if (e.stage === 'failed') reject(new Error(e.error?.message))
    }
  }
)
```

В браузере `EventSource` тоже работает (через session-cookie + `withCredentials: true`), но не поддерживает custom headers — для API-key используйте `fetchEventSource` или `fetch` с `ReadableStream`.

## Heartbeat-фреймы

Каждые `SSE_HEARTBEAT_MS` мс (default 15000):

```
: ping
```

Comment-фрейм для удержания соединения через reverse-proxy. Клиент игнорирует.

## Подключение после завершения

Можно подключиться **после** terminal-стадии: gateway отдаст последнее событие и закроет стрим. Удобно для refresh страниц с сохранённым `requestId`.

## Лимит

`SSE_MAX_CONNECTIONS_PER_USER` (default 5). Превышение:

```
HTTP/1.1 429 Too Many Requests
Retry-After: 60
{"error":"rate_limited","code":"too_many_sse_connections"}
```

Слот освобождается при close либо TTL 1 час.

## Альтернатива — webhook

| Используйте webhook когда | Причина |
| --- | --- |
| Видео > 5 мин | Мобильная сеть закроет SSE |
| Серверный pipeline без UI | Некому держать соединение |
| Нужны retries при ошибке доставки | SSE не ретраится |

См. [формат webhooks](/reference/webhooks).
