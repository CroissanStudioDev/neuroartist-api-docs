# Формат webhooks

Push-уведомление о завершении генерации на ваш HTTPS-URL. Альтернатива polling и SSE для серверных pipeline-ов.

## Когда использовать

| Хочу | Решение |
| --- | --- |
| UI в реальном времени в браузере | [SSE](/reference/sse) |
| Серверный pipeline без активного клиента | **Webhook** |
| Видео-генерация / batched job | **Webhook** |
| Повторные попытки при ошибке доставки | **Webhook** (6 попыток с backoff) |

## Подключение

1. Получите webhook secret: `POST /me/webhook-secret/rotate` (session-only).
2. Подготовьте HTTPS-endpoint, отвечающий `200 OK`.
3. Передавайте `?webhook=<URL>` при отправке задачи:

```bash
curl -X POST 'https://api.neuroartist.ru/queue/fal-ai/flux/dev?webhook=https://your.app/cb' \
  -H "Authorization: Bearer na_live_xxxx" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"…"}'
```

## Заголовки и тело

```
POST /cb HTTP/1.1
Content-Type: application/json
Webhook-Id: msg_abc123
Webhook-Timestamp: 1714050000
Webhook-Signature: v1,base64-encoded-hmac-sha256
```

Тело для completed:

```json
{
  "type": "generation.completed",
  "requestId": "req_abc",
  "modelId": "fal-ai/flux/dev",
  "userId": "u_…",
  "completedAt": "…",
  "result": { "images": [{"url":"https://s3.cloud.ru/…"}], "seed": 12345 }
}
```

Тело для failed:

```json
{
  "type": "generation.failed",
  "requestId": "req_abc",
  "modelId": "…",
  "failedAt": "…",
  "error": { "code": "provider_error", "message": "…" }
}
```

URL-ы в `result` уже переписаны на S3.

## Валидация

Подпись Svix-style — над `{webhook_id}.{timestamp}.{body}`:

```js
import crypto from 'node:crypto'

function verify(req, secret) {
  const id = req.headers['webhook-id']
  const ts = req.headers['webhook-timestamp']
  const sig = req.headers['webhook-signature']

  if (Math.abs(Date.now() / 1000 - Number(ts)) > 300) return false  // anti-replay 5 мин

  const key = Buffer.from(secret, 'base64')
  const expected = 'v1,' + crypto.createHmac('sha256', key)
    .update(`${id}.${ts}.${req.rawBody}`).digest('base64')

  return sig.split(' ').some(s =>
    crypto.timingSafeEqual(Buffer.from(s), Buffer.from(expected))
  )
}
```

**Никогда** не сравнивайте подписи через `===` — timing-атака. Только `timingSafeEqual` / `compare_digest` / `hash_equals`.

## Идемпотентность

Webhook может прийти повторно. Дедупайте по `webhook-id`:

```js
const seen = await redis.set(`wh:${id}`, '1', 'NX', 'EX', 86400)
if (seen === null) return res.status(200).end()
```

## Повторные попытки

| Попытка | Задержка |
| --- | --- |
| 2 | 1 минута |
| 3 | 5 минут |
| 4 | 30 минут |
| 5 | 2 часа |
| 6 | 5 часов |

После 6-й попытки доставка прекращается, реквест помечается `webhook_dead` в `/me/activity`.

## Внутренние webhooks (НЕ для вас)

`/webhooks/{providerId}` и `/webhooks/yookassa/notify` — это **входящие** server-to-server endpoints между gateway и upstream-ами (fal.ai, YooKassa). В клиентский код к ним обращаться не нужно.
