# `/webhooks`

<Callout type="info">
  Это **входящие** webhooks — server-to-server, между upstream-ами и gateway. Клиентский код к ним обращаться не должен. Если вы интегрируете SDK / приложение, см. [Concepts → Webhooks](/reference/webhooks) — там про **исходящие** webhooks gateway → ваш сервис.
</Callout>

## `POST /webhooks/{providerId}`

Webhook от ML-провайдера (fal.ai, replicate, azure).

**Защита:** ED25519-подпись провайдера через JWKS. В dev можно отключить через `FAL_WEBHOOK_DEV_BYPASS=true` (только при `NODE_ENV=development`).

**Path:** `providerId` ∈ {`fal`, `replicate`, `azure`} (registry зависит от установленных провайдеров).

**Body:**

```json
{
  "request_id": "req_abc",
  "status": "OK",
  "error": null,
  "payload": { … },
  "payload_error": null
}
```

`status` — `OK` или `ERROR`.

**Response 200:**

```json
{
  "ok": true,
  "status": "completed",
  "deduped": false,
  "assets": { "found": 1, "uploaded": 1, "failed": 0 }
}
```

**Errors:**

- `400 invalid_json` / `missing_request_id`.
- `401 invalid_signature`.
- `501 provider_not_implemented`.

**Rate limit:** 100 / 1 минута / IP.

## `POST /webhooks/yookassa/notify`

Webhook от YooKassa о состоянии платежа.

**Защита:** IP-whitelist (CIDR YooKassa) + HMAC. В dev можно отключить через env.

**Body** (форматы YooKassa):

```json
{
  "event": "payment.succeeded",
  "object": {
    "id": "2c…",
    "status": "succeeded",
    "amount": { "value": "1000.00", "currency": "RUB" },
    "metadata": { "userId": "u_…", "creditsToGrant": "1000" },
    "payment_method": { "id": "pm_abc", "type": "bank_card", "card": { "last4": "4242" } }
  }
}
```

**Response 200:** `{ "ok": true, "processed": true }` — или `{ "processed": false, "reason": "duplicate" }` при дедупе.

**Идемпотентность:** Redis SETNX по `webhook_id` с TTL 1 час.

**Что делает:**

- На `payment.succeeded`: зачисляет кредиты, сохраняет `payment_method` в `payment_method` таблицу (если ещё нет), отправляет email-нотификацию.
- На `payment.canceled`: помечает платёж `canceled`, кредиты не зачисляются.

**Errors:** `400 invalid_json` / `invalid_body`. Подпись/IP — игнорируются (200 OK с `processed: false, reason: …`), чтобы YooKassa не ретраила бесконечно.
