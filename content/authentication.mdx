# Аутентификация

Любая защищённая ручка принимает **один из**:

```
Authorization: Bearer na_live_xxxxxxxxxxxxxxxx
x-api-key: na_live_xxxxxxxxxxxxxxxx
Cookie: better-auth.session_token=…
```

## Когда что использовать

| Сценарий | Способ |
| --- | --- |
| Серверный код, CLI, бэкенд-интеграция | API-ключ |
| Браузер, дашборд | Session cookie |
| Управление ключами / платежами / профилем | **Только** session (см. ниже) |

## API-ключ

Создание — UI или `POST /api/auth/api-key/create`. Формат: `na_live_` + 32 символа. Возвращается **один раз** при создании.

```bash
curl https://api.neuroartist.ru/me -H "Authorization: Bearer na_live_xxxxxxxxxxxxxxxx"
```

```json
{
  "user": { "id": "u_…", "email": "you@example.com", "role": "user" },
  "auth": { "source": "apiKey", "apiKeyId": "key_abc" }
}
```

### Месячный лимит на ключ

```bash
curl -X PATCH https://api.neuroartist.ru/me/keys/key_abc \
  -H "Content-Type: application/json" -b cookies.txt \
  -d '{"metadata":{"monthlyLimitRub":5000}}'
```

Превышение → `429 key_limit_exceeded`. Сбрасывается 1 числа.

### Session cookie

Управляются [Better Auth](https://better-auth.com/) под `/api/auth/*`.

```bash
# Регистрация
curl -X POST https://api.neuroartist.ru/api/auth/sign-up/email \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"…","name":"You"}' \
  -c cookies.txt

# Дальше — все запросы с -b cookies.txt
curl https://api.neuroartist.ru/me -b cookies.txt
```

Cookie: `better-auth.session_token`, `HttpOnly; Secure; SameSite=Lax`. Cross-origin браузерные запросы — с `credentials: 'include'`.

### Yandex OAuth

```
GET /api/auth/sign-in/social?provider=yandex
```

Callback — `/api/auth/callback/yandex`.

## Что строго session-only

API-ключ принимается везде, кроме операций, где утечка ключа дала бы критический ущерб:

- `PATCH /me`, `/me/sessions`, `/me/keys`, `/me/payment-methods/*`, `/me/auto-topup`, `/me/webhook-secret`, `/me/uploads`
- `POST /billing/topup`, `GET /billing/payments/{id}/receipt`
- Все `/admin/*`

Эти ручки вернут `403 forbidden` для запросов с API-key.

## Практики для production

- Отдельный ключ на каждое окружение (`prod`, `staging`, `local`).
- Хранить в env / secrets manager. Не в репо.
- Ротация при компрометации: `DELETE /me/keys/{id}` + создать новый.
- Активные сессии — `GET /me/sessions`, ревок — `DELETE /me/sessions/{id}`.
