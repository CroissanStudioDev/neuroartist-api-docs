# Авто-пополнение

Чтобы не получать `402 insufficient_balance` посреди pipeline.

```
balance < thresholdRub  →  charge amountRub  →  webhook  →  +amountRub в balance
```

Срабатывает в момент попытки списания за генерацию. Не чаще раза в 60 секунд.

## Подключение

```bash
# 1. Один раз заплатить через UI — gateway сохранит payment_method
# (Top-up через дашборд /billing/topup; YooKassa возвращает payment_method_id, gateway его сохраняет)

# 2. Сделать карту default
curl -X POST https://api.neuroartist.ru/me/payment-methods/pm_abc/default \
  -b cookies.txt

# 3. Включить auto-topup
curl -X PATCH https://api.neuroartist.ru/me/auto-topup \
  -H "Content-Type: application/json" -b cookies.txt \
  -d '{"enabled":true,"thresholdRub":200,"amountRub":1000}'
```

`thresholdRub` ≥ `AUTO_TOPUP_MIN_THRESHOLD_RUB` (default 10).
`amountRub` ∈ `[AUTO_TOPUP_MIN_AMOUNT_RUB, AUTO_TOPUP_MAX_AMOUNT_RUB]` (default 50–10 000).

## Выключить

```bash
curl -X PATCH https://api.neuroartist.ru/me/auto-topup \
  -H "Content-Type: application/json" -b cookies.txt \
  -d '{"enabled":false}'
```

Backend очищает `thresholdRub` и `amountRub`. Чтобы включить заново, передайте оба значения снова.

## Профили

| Профиль | thresholdRub | amountRub |
| --- | --- | --- |
| Hobby | 50 | 200 |
| Production-сервер | 200 | 1 000 |
| Heavy pipeline (видео) | 500 | 5 000 |

## Если auto-topup упал

(карта отклонена, нет default метода) — обычная ошибка `402 insufficient_balance` на текущем submit. Email + запись `topup_failed` в `/me/activity`.

## Альтернатива — cron

Если нужна более сложная логика (расписание, разные карты, batch-ы) — поднимите свой scheduled job, дёргающий `POST /billing/topup`. Авто-пополнение только реактивное.
