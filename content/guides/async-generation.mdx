# Async-генерация через `/queue` + SSE

Async-режим даёт:

- Мгновенный ответ с `requestId` (без ожидания terminal-состояния).
- Реалтайм-прогресс через SSE.
- Возможность держать сразу несколько генераций «в работе».
- Отмену незавершённой генерации.

## Полный цикл

<Steps>

### Submit

```bash
curl -X POST https://api.neuroartist.ru/queue/fal-ai/flux/dev \
  -H "Authorization: Bearer na_live_xxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"editorial portrait","image_size":"portrait_4_3"}'
```

```json
{
  "requestId": "req_abc",
  "status": "IN_QUEUE",
  "queuePosition": 1
}
```

Кредиты уже списаны атомарно. Сохраните `requestId`.

### Подпишитесь на прогресс

```js
import { fetchEventSource } from '@microsoft/fetch-event-source'

await fetchEventSource(
  `https://api.neuroartist.ru/queue/fal-ai/flux/dev/requests/${requestId}/progress/stream`,
  {
    headers: { Authorization: `Bearer ${NA_API_KEY}` },
    onmessage(msg) {
      const event = JSON.parse(msg.data)
      console.log(event.stage, event)

      if (event.stage === 'completed') {
        console.log('Done:', event.result)
      }
      if (event.stage === 'failed') {
        console.error('Failed:', event.error)
      }
    }
  }
)
```

Стрим автоматически закроется после `completed`/`failed`. Подробнее — [SSE](/concepts/sse).

### Заберите результат (если нужно)

В большинстве кейсов событие `completed` уже содержит весь payload. Но можно перечитать:

```bash
curl 'https://api.neuroartist.ru/queue/fal-ai/flux/dev/requests/req_abc' \
  -H "Authorization: Bearer na_live_xxxxxxxxxxxxxxxx"
```

### Отмена (опционально)

```bash
curl -X PUT 'https://api.neuroartist.ru/queue/fal-ai/flux/dev/requests/req_abc/cancel' \
  -H "Authorization: Bearer na_live_xxxxxxxxxxxxxxxx"
```

Возможна только до завершения. Кредиты возвращаются.

</Steps>

## Альтернатива SSE — polling

Если SSE не подходит (например, в окружении, где нельзя держать долгое соединение):

```bash
# Опрос статуса каждые 2 секунды
while true; do
  STATUS=$(curl -s 'https://api.neuroartist.ru/queue/fal-ai/flux/dev/requests/req_abc/status' \
    -H "Authorization: Bearer na_live_xxxxxxxxxxxxxxxx" | jq -r .status)
  echo "$STATUS"
  [[ "$STATUS" == "COMPLETED" || "$STATUS" == "FAILED" ]] && break
  sleep 2
done

# Забор результата
curl 'https://api.neuroartist.ru/queue/fal-ai/flux/dev/requests/req_abc' \
  -H "Authorization: Bearer na_live_xxxxxxxxxxxxxxxx"
```

Учтите, что это съедает квоту API-key (1000 req/h) сильнее, чем один SSE-стрим.

## Альтернатива SSE — webhook

Самый правильный путь для серверных пайплайнов:

```bash
curl -X POST 'https://api.neuroartist.ru/queue/fal-ai/flux/dev?webhook=https://your.app/api/na-callback' \
  -H "Authorization: Bearer na_live_xxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"editorial portrait"}'
```

После `completed`/`failed` gateway POST-ит JSON на ваш URL с подписью. Подробнее — [Webhooks](/concepts/webhooks).

## Параллельные генерации

Можно держать сразу несколько submit-ов «в работе»:

```js
const requestIds = await Promise.all([
  submit({ prompt: 'red flower' }),
  submit({ prompt: 'blue flower' }),
  submit({ prompt: 'yellow flower' })
])

// Подписаться на 3 SSE сразу — норм, лимит 5 на юзера
const results = await Promise.all(
  requestIds.map(id => waitForCompletion(id))
)
```

Лимит — `SSE_MAX_CONNECTIONS_PER_USER` (по умолчанию 5). Превышение → `429 too_many_sse_connections`.

## Смешанный паттерн (recommended)

В проде часто оптимально:

1. **Submit + webhook** для серверной части (idempotent, переживает рестарты).
2. **SSE** для активного UI клиента (опционально, если есть открытая вкладка).

Так клиент видит прогресс в реальном времени, а сервер всё равно получит финал даже если клиент закроется.

## Late-join — подключение к уже завершённой генерации

Если SSE-клиент монтируется **после** того, как генерация уже завершилась — gateway сразу отдаст последнее событие и закроет стрим:

```js
// Эта генерация уже completed 30 минут назад
const requestId = 'req_abc'
const es = new EventSource(`.../progress/stream`)
es.onmessage = (e) => {
  const event = JSON.parse(e.data)
  // Получите stage: 'completed' и result
  console.log(event)
  es.close()
}
```

Это сильно упрощает код перезагрузки страниц с сохранённым `requestId`.

## Когда `?passthrough=1`

Чтобы получить **сырые URL-ы провайдера** (без переписывания на S3):

```bash
curl 'https://api.neuroartist.ru/queue/fal-ai/flux/dev/requests/req_abc?passthrough=1' \
  -H "Authorization: Bearer na_live_xxxxxxxxxxxxxxxx"
```

Используйте только для отладки или если у вас свой механизм re-hosting. Иначе ссылки умрут через ~24 часа.

## Полный код полного цикла

```js
import { fetchEventSource } from '@microsoft/fetch-event-source'

const NA_API_KEY = process.env.NA_API_KEY

async function runAsync(modelId, input) {
  // 1. Submit
  const submit = await fetch(
    `https://api.neuroartist.ru/queue/${modelId}`,
    {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${NA_API_KEY}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(input)
    }
  ).then(r => r.json())

  if (!submit.requestId) {
    throw new Error(`Submit failed: ${submit.message}`)
  }

  // 2. Wait via SSE
  return new Promise((resolve, reject) => {
    fetchEventSource(
      `https://api.neuroartist.ru/queue/${modelId}/requests/${submit.requestId}/progress/stream`,
      {
        headers: { Authorization: `Bearer ${NA_API_KEY}` },
        onmessage(msg) {
          const event = JSON.parse(msg.data)
          if (event.stage === 'completed') resolve(event.result)
          if (event.stage === 'failed') reject(new Error(event.error?.message))
        },
        onerror(err) {
          reject(err)
        }
      }
    )
  })
}

const result = await runAsync('fal-ai/flux/dev', {
  prompt: 'editorial portrait',
  image_size: 'portrait_4_3'
})

console.log(result.images[0].url)
```
