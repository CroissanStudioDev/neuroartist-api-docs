# Приём webhooks

Полная реализация приёмника на Node.js (Hono / Express / Next.js Route Handler) и Python (FastAPI). Концептуальное описание формата — в [Webhooks](/concepts/webhooks).

## Чек-лист

<Steps>

### 1. Получите webhook secret

```bash
# Один раз при настройке
curl https://api.neuroartist.ru/me/webhook-secret -b cookies.txt
# или, если ещё не создан:
curl -X POST https://api.neuroartist.ru/me/webhook-secret/rotate -b cookies.txt
```

Сохраните в env: `NA_WEBHOOK_SECRET=wsec_…`.

### 2. Поднимите endpoint, доступный по HTTPS

Любой роут, отвечающий `200 OK` на `POST`. Localhost не подойдёт — для dev используйте `ngrok` или `cloudflared`:

```bash
ngrok http 3000
# https://abcd-1234.ngrok-free.app/api/na-webhook
```

### 3. Передавайте URL в submit

```bash
curl -X POST 'https://api.neuroartist.ru/queue/fal-ai/flux/dev?webhook=https://abcd.ngrok-free.app/api/na-webhook' \
  -H "Authorization: Bearer na_live_xxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"…"}'
```

</Steps>

```ts
import { Hono } from 'hono'
import crypto from 'node:crypto'

const app = new Hono()
const SECRET = process.env.NA_WEBHOOK_SECRET!.replace(/^wsec_/, '')

app.post('/api/na-webhook', async (c) => {
  const id = c.req.header('webhook-id')
  const ts = c.req.header('webhook-timestamp')
  const sig = c.req.header('webhook-signature')
  if (!id || !ts || !sig) return c.json({ ok: false }, 400)

  // 1. Защита от replay (5 минут)
  if (Math.abs(Date.now() / 1000 - Number(ts)) > 300) {
    return c.json({ ok: false }, 401)
  }

  // 2. Подпись
  const raw = await c.req.text()
  const key = Buffer.from(SECRET, 'base64')
  const expected = 'v1,' + crypto
    .createHmac('sha256', key)
    .update(`${id}.${ts}.${raw}`)
    .digest('base64')

  const valid = sig.split(' ').some(s => {
    try { return crypto.timingSafeEqual(Buffer.from(s), Buffer.from(expected)) }
    catch { return false }
  })
  if (!valid) return c.json({ ok: false }, 401)

  // 3. Идемпотентность
  const seen = await redis.set(`na:wh:${id}`, '1', { NX: true, EX: 86400 })
  if (seen === null) return c.json({ ok: true, deduped: true })

  // 4. Обработка
  const event = JSON.parse(raw)
  if (event.type === 'generation.completed') {
    await onCompleted(event)
  } else if (event.type === 'generation.failed') {
    await onFailed(event)
  }

  return c.json({ ok: true })
})
```

```js
import express from 'express'
import crypto from 'node:crypto'

const app = express()
const SECRET = process.env.NA_WEBHOOK_SECRET.replace(/^wsec_/, '')

// IMPORTANT: raw body, не JSON-middleware
app.post('/api/na-webhook',
  express.raw({ type: 'application/json' }),
  (req, res) => {
    const id = req.header('webhook-id')
    const ts = req.header('webhook-timestamp')
    const sig = req.header('webhook-signature')
    if (!id || !ts || !sig) return res.status(400).end()

      return res.status(401).end()

      catch { return false }
    })
```

```python
import os
import hmac
import base64
import time
import json
from fastapi import FastAPI, Request, HTTPException

app = FastAPI()
SECRET = os.environ["NA_WEBHOOK_SECRET"].removeprefix("wsec_")

@app.post("/api/na-webhook")
async def na_webhook(req: Request):
    wid = req.headers.get("webhook-id")
    wts = req.headers.get("webhook-timestamp")
    wsig = req.headers.get("webhook-signature")
    if not (wid and wts and wsig):
        raise HTTPException(400)

    if abs(time.time() - int(wts)) > 300:
        raise HTTPException(401)

    raw = (await req.body()).decode()
    key = base64.b64decode(SECRET)
    expected = "v1," + base64.b64encode(
        hmac.new(key, f"{wid}.{wts}.{raw}".encode(), "sha256").digest()
    ).decode()

    if not any(hmac.compare_digest(s, expected) for s in wsig.split(" ")):
        raise HTTPException(401)

    event = json.loads(raw)
    # process…

    return {"ok": True}
```

## Re-trigger после ошибки

Если ваш приёмник упал и вернул 5xx — gateway сам ретраит до 6 раз с экспоненциальным backoff. Никаких действий с вашей стороны не нужно — поправьте код, дождитесь следующей попытки.

Если все 6 попыток сгорели — реквест помечается `webhook_dead` в `/me/activity`. Можно поднять руками через `POST /me/activity/{requestId}/replay-webhook` (если эта ручка включена в вашем тарифе) либо просто скачать результат через `/queue/.../result`.

## Чего НЕ делать

- ❌ Не парсите body **до** проверки подписи. Используйте raw bytes.
- ❌ Не сравнивайте подписи через `===`. Только `timingSafeEqual` / `compare_digest` / `hash_equals`.
- ❌ Не игнорируйте `webhook-timestamp` — иначе уязвимы к replay.
- ❌ Не делайте долгую работу синхронно в обработчике. Получили событие → положили в очередь → ответили `200`. Иначе уйдёте за timeout-ы.
